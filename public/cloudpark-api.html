Integração CloudPark

Bem-vindo ao guia oficial de integração do sistema CloudPark com o ecossistema Leve ERP. Aqui você encontrará todas as definições técnicas, schemas e exemplos práticos para alimentar o faturamento em tempo real através de Webhooks.

Segurança e Autenticação

A nossa API utiliza o padrão **HTTP Bearer Auth**. Todas as requisições devem incluir o cabeçalho Authorization para serem processadas.

Bearer Token dc952bbb65a31755a53a830233cfe452bd4feca4f42add52e26a3b8a3babd9e0

Endpoints

POST /api/integracoes/cloudpark/receber-movimentos

Este endpoint recebe um lote de movimentos financeiros e os processa imediatamente para consolidação diária.

Especificação do Payload (JSON)

Campo Tipo Descrição
filial Integer Código identificador da filial da CloudPark. Deve estar mapeado no Leve ERP.
items[] Array Lista de transações individuais a serem processadas.
↳ payday DateTime Data e hora do pagamento. Formato ISO 8601 (ex: 2024-03-31T19:57:45Z).
↳ category String Categoria do movimento (ex: "AVULSO", "MENSALISTA").
↳ value Decimal Valor monetário total da transação.
↳ payment_method String Modo de pagamento (ex: "Dinheiro", "Cartão de crédito", "Pix").

Exemplos de Implementação

curl -X POST "https://dashboard.levemobilidade.com.br/api/integracoes/cloudpark/receber-movimentos" \
-H "Authorization: Bearer dc952bbb65a31... \
-H "Content-Type: application/json" \
-d '{
  "filial": 98,
  "items": [{
    "payday": "2024-03-31T19:57:45Z",
    "value": 13.0,
    "payment_method": "Pix"
  }]
}'
# Envio via PowerShell
$headers = @{ "Authorization" = "Bearer dc952b..." }
Invoke-RestMethod -Uri "https://dashboard.levemobilidade.com.br/api/integracoes/cloudpark/receber-movimentos" `
    -Method Post `
    -Headers $headers `
    -ContentType "application/json" `
    -Body '{ "filial": 1141, "items": [...] }'

Taxas e Limites

  • Rate Limit 60 req/min
  • Batch Limit 50 items/req
  • Max Body 2MB

Deduplicação

Nossa API utiliza um hash SHA-256 (Baseado em Filial + Payday + Value + Description). Caso um movimento processado anteriormente seja enviado novamente, ele será **ignorado** pelo sistema para evitar erros contábeis.
